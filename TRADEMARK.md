# Trademark Policy

The **openZro** name and logo are trademarks of the openZro project
maintainers and are governed by this policy. The project's source code
is released under [BSD-3-Clause](LICENSE), which is a copyright
license. **The BSD-3 license does NOT grant trademark rights.** This
document is the only authority on use of the openZro marks.

## What this protects

- The word mark **openZro** (in any visual styling, including the
  signature `open` + bold violet `Z` + `ro` wordmark).
- The openZro icon (`brand/openzro-icon.svg`) and the icon + wordmark
  lockup.
- The `openzro.io` domain and `openZro` GitHub organization name.
- Common derivations such as **openzro**, **OpenZro**, **OPENZRO**,
  **open-zro**, etc., when used to identify software, services or
  events.

## What you may do without asking

You may freely use the openZro name to **describe, discuss, review,
teach, link to, integrate with, or critique** the project. Specific
permitted uses include:

- "Our team uses openZro for internal access" — factual use.
- "How to set up openZro on Kubernetes" — content about the project.
- "openZro plugin for Foo" — naming an unofficial integration, **as
  long as** it's clear the plugin is community-built and not endorsed
  by the openZro maintainers (a disclaimer in the README is enough).
- Distribution of unmodified openZro binaries downloaded from the
  official sources, retaining the openZro name.
- Reference in books, articles, conference talks, podcasts, blog posts.

## What requires written permission

You **must** request permission from the maintainers (open a
discussion at https://github.com/openzro/openzro/discussions/categories/trademark
or email `trademark@openzro.io`) before doing any of the following:

- Naming a fork "openZro" or with a "openZro"-derived name.
  Acceptable: "Forked from openZro, called Foo". Not acceptable:
  "openZro-Pro", "openZro Plus", "openZro Cloud".
- Distributing **modified** builds under the openZro name. If you
  change the code and ship it, rename it. You may still say "based
  on openZro" in the description.
- Using the openZro logo or wordmark in commercial marketing
  material, paid advertisements, product packaging, or
  trade-show booths.
- Naming a paid commercial service, SaaS, or appliance "openZro
  Hosting", "openZro Enterprise" or similar.
- Registering domain names, GitHub orgs, npm/Docker namespaces,
  trademark applications or social-media handles that include the
  openZro mark.

## What is never allowed

- Implying that an unofficial fork or third-party service is the
  **official** openZro project.
- Using the openZro marks in a way that could confuse users about
  the source of software or services.
- Using the openZro marks in connection with malware, scams,
  phishing, illegal activity, or content that violates the
  [Code of Conduct](CODE_OF_CONDUCT.md).
- Filing a trademark registration for the openZro mark (or a
  confusingly similar variant) in any jurisdiction.

## Forks of the code

The BSD-3 license permits forking the code under any name, including
non-openZro names. You may keep references to "openZro" inside source
comments and `LICENSE` / `AUTHORS` attribution. You may **not** keep
"openZro" as the name of your fork, its binaries, its container
images, its package names, or its public branding.

## Reporting misuse

If you see the openZro marks being used in a way that violates this
policy — particularly in scams, phishing pages, or unauthorized
commercial offerings — please report it by opening a GitHub issue
with the `trademark` label or emailing `trademark@openzro.io`. The
maintainers will evaluate and act, up to and including takedown
requests, DMCA notices, and legal action where warranted.

## Why we have this policy

Open-source software needs a clear, recognizable name so operators
know what they're running and where to get support. A trademark policy
keeps the openZro name a **reliable signal** of what you're getting,
while leaving the code itself fully free for anyone to fork, modify and
ship under a different name. This is the same model used by Kubernetes,
Prometheus, Linux, Mozilla, and most major open-source projects.
